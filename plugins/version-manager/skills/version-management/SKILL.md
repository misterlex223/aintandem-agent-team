# 版本管理技能

這個技能提供自然語言介面來使用版本管理功能。

## 觸發關鍵字

當用戶提到以下內容時觸發：

- 「發布版本」、「準備發布」、「release」
- 「版本管理」、「version management」
- 「建議版本號」、「什麼版本號」
- 「版本狀態」、「目前版本」、「當前版本」
- 「版本歷史」、「版本對比」、「版本差異」
- 「建立快照」、「版本快照」
- 「完成發布」、「finalize release」

## 使用方式

### 準備發布

```
用戶: 幫我準備發布 v0.7.0
你: 我將使用 version-manager 來引導你完成版本發布的準備工作。
     [執行 prepare-release 流程]
```

### 版本狀態

```
用戶: 目前版本是什麼？
你: 讓我查詢當前版本狀態。
     [執行 version-status 流程]
```

### 建議版本號

```
用戶: 你覺得下個版本號應該是什麼？
你: 我來分析 PRD 變更並建議版本號。
     [執行 suggest-version 流程]
```

### 版本對比

```
用戶: 對比 v0.6.0 和 v0.7.0 的差異
你: 我將對比這兩個版本的規格差異。
     [執行 version-history 流程]
```

## 工作流程

1. **識別意圖** - 判斷用戶想做什麼
2. **提取參數** - 版本號、對比目標等
3. **呼叫對應功能** - prepare-release / version-status / suggest-version / version-history
4. **顯示結果** - 以友善的格式呈現

## 範例對話

### 範例 1：準備發布

```
用戶: 我們準備發布新版本了

你: 好的！讓我先分析 PRD 的變更內容。

     [讀取 PRD.md]

     我發現了 3 個新功能：
     - 批次上傳多張圖片
     - 自訂品牌覆蓋層
     - 多種價格層級

     根據 Semantic Versioning，這是 MINOR 版本更新。
     建議版本號：v0.7.0（當前：v0.6.0）

     是否採用此版本號並開始準備發布？

用戶: 是的

你: 好的，我將引導你完成發布檢查清單。
     [執行 prepare-release 流程]
```

### 範例 2：查詢狀態

```
用戶: 現在開發進度如何？

你: 讓我查詢版本狀態。

     📍 當前版本: v0.6.0
     🎯 下個版本: v0.7.0 (minor)
     📊 狀態: in-development

     ✅ 功能進度: 2/3 完成
       ✅ 2.8 批次上傳
       ✅ 2.9 品牌覆蓋層
       ⏳ 2.10 價格層級

     還有一個功能未完成，需要協助嗎？
```

### 範例 3：版本對比

```
用戶: v0.6.0 和 v0.7.0 有什麼差別？

你: 讓我對比這兩個版本的差異。

     [讀取兩個版本的 PRD.md]

     ## v0.6.0 → v0.7.0 變更摘要

     ✨ 新增功能:
       - 批次上傳多張圖片
       - 自訂品牌覆蓋層
       - 多種價格層級

     🔄 修改內容:
       - 2.2 價格覆蓋層：新增多價格支援

     ❌ 移除功能: 無

     整體來說，v0.7.0 主要在提升效率和品牌功能。
```

## 與 Command 的差異

- **Command (`/version-manager`)**: 快速執行標準操作
- **Skill (自然語言)**: 對話式引導，更靈活

建議：
- 熟練用戶使用 Command
- 不確定如何操作時用自然語言
